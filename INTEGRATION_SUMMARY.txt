│ • Calculates reliability % per cluster                                       │

│ • Prioritizes for upgrades                                                   │

│ • Mean Time To Restore by hour                                               │
│ • Peak risk identification                                                    │
│ Result: 14:00 & 21:00 are critical risk hours                               │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ InfrastructureOptimizer ──────────────────────────────────────────────────┐

🚀 HOW TO USE
═══════════════════════════════════════════════════════════════════════════════

1. Start the application:
   $ npm start
   (Backend on :8000, Frontend on :5173)

2. Go to Dashboard → upload CSV files or use existing data

3. Click "🔬 Napredna Analitika" tab

4. Choose dataset (vsi_podatki, ovrednoteni, or uploads)

5. Select analysis type:
   • Event Reconstruction - See network-wide patterns
   • Geographic Clustering - Identify infrastructure hotspots
   • Hourly Risk Model - Predict when outages likely occur
   • Infrastructure Roadmap - Plan upgrades strategically
